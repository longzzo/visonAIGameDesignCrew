# Vision Engine 웹앱

게임 기획 전용 멀티에이전트 웹앱. OpenClaw 게이트웨이(ws://127.0.0.1:18789)에 WebSocket으로 연결해
8개 기획 에이전트(PM·시나리오·게임플레이·시스템·UI/UX·밸런스·BM·아트)를 부리고,
산출물을 마스터 GDD(`../workspace/GDD.md`)에 실시간 반영한다.

## 실행

```bash
# 사전 조건: Ollama 실행 중 + OpenClaw 게이트웨이 실행 중
#   ollama serve                      (또는 트레이 앱, 모델경로 D:\Ollama\models)
#   openclaw daemon status            (stopped면: openclaw daemon start)

cd vision-engine/webapp
npm install        # 최초 1회
npm run dev        # http://127.0.0.1:5199
```

`npm run dev` 는 먼저 `scripts/prepare-env.mjs` 를 실행해 `~/.openclaw/openclaw.json` 의
게이트웨이 토큰을 `.env.local` 로 가져온다(커밋 금지 파일).

## 화면 구성

| 영역 | 내용 |
|---|---|
| 상단 바 | 프로젝트명(수정 가능) · 뷰 전환 · 모델 표시 · 토큰 카운터(로컬=\$0) · 연결 상태 |
| 왼쪽 사이드바 | 8개 에이전트 목록 + 상태 표시등(대기/작업 중/완료/오류) |
| 중앙 (오케스트레이션 뷰) | 요청 입력 → 선택한 에이전트들에게 분배 → 카드로 진행 상황 실시간 표시 → PM이 개요 통합 |
| 중앙 (채팅 뷰) | 선택한 에이전트와 1:1 대화 (스트리밍) |
| 오른쪽 패널 | 마스터 GDD 실시간 미리보기 · 섹션 이동 · 전체 편집 · 드래그앤드롭 반영 |

## 동작 방식 (오케스트레이션)

1. 요청 입력 → 선택된 전문 에이전트들에게 순차/병렬(1~2) 팬아웃 (`agent` 메서드, 에이전트별 독립 세션)
2. 각 산출물은 완료 즉시 GDD의 담당 섹션에 자동 반영(끌 수 있음)
3. 모든 전문 에이전트 완료 후 PM이 "1. 개요" 섹션을 통합 작성
4. GDD 저장은 웹앱(Vite 미들웨어 `/api/gdd`)이 수행 — 로컬 8B 모델의 툴콜링에 의존하지 않는 안정 설계

## 제한 사항

- 로컬 qwen3:8b 기준 에이전트 1명당 30초~2분 소요. 동시 실행 2는 VRAM 8GB에서 느려질 수 있어 기본값 1.
- 토큰 카운터는 게이트웨이가 사용량을 안 줄 때 문자 수 기반 추정치(≈ 표시)로 대체.
- 소형 모델 특성상 가끔 도구 호출 JSON이 텍스트로 새어 나옴 → 웹앱이 자동으로 내용만 추출·정리.
- 게이트웨이 중단 시: `openclaw daemon start` (일반 터미널에서 안 되면 `vision-engine/bin/openclaw.cmd` 사용)
