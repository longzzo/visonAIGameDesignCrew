# 🎮 기존 유니티 프로젝트 리뷰 — 셋업 가이드

이미 진행 중이거나 완성된 유니티 프로젝트를 개발팀에게 맡겨 **코드 리뷰·구조 점검·보완점**을 받는 기능입니다.
경로 하나만 등록하면 나머지는 자동입니다.

---

## 1. 가장 간단한 방법 (파일 읽기 리뷰) — 준비물 없음에 가까움

개발팀이 프로젝트 파일을 **직접 읽어** 아키텍처·코드 품질·성능 관점의 보완점을 뽑습니다.
유니티 에디터를 켤 필요도 없습니다.

1. 앱 실행 후 사이드바 **🔌 MCP 도구 서버** 를 엽니다.
2. 맨 위 **🎮 기존 유니티 프로젝트 리뷰** 칸에 프로젝트 **루트 폴더 경로**를 넣습니다.
   - 루트 = `Assets` 폴더가 들어 있는 그 폴더 (예: `D:\MyGame`)
3. **연결** 클릭 → 🟢 연결됨 이 뜨면 준비 끝.
4. 사무실 → **🛠️ 2F 개발팀** → 상단 **🎮 유니티 프로젝트 리뷰** 버튼 클릭.
   - 선임 개발자가 프로젝트를 훑고(구현 리드), 코드 리뷰어가 검토하고, 테스트 엔지니어가 검증하는
     **개발 회의(교차 검증)**가 자동으로 돌아갑니다.
   - 결과는 그 자리에서 단계별로 보이고, 원하면 지시문을 바꿔 다시 돌릴 수 있습니다.

> 내부적으로: 등록한 경로에 파일시스템 MCP 서버(`unity-project`)가 자동 연결되고, 개발팀 8명에게
> 그 도구가 배정됩니다. 이 서버는 **읽기·쓰기 모두 가능**하니, 진짜 프로젝트를 넘기기 전에
> 사본이나 git으로 백업된 상태에서 쓰는 걸 권장합니다.

---

## 2. 유니티 에디터를 실제로 조종하려면 (씬·플레이모드·C# 실행)

파일 읽기를 넘어 **에디터 안에서** 씬을 만지고, 플레이모드를 켜고, 스크린샷을 찍고, C#을 실행하려면
아래 유니티 MCP 서버 중 하나를 설치합니다. 팀별로 이미 배정돼 있습니다:

| 팀 | 서버 | 저장소 | 하는 일 |
|---|---|---|---|
| 개발 구현팀 | `unity-coplay` | [CoplayDev/unity-mcp](https://github.com/CoplayDev/unity-mcp) | 에셋·씬·스크립트·테스트 (47도구, MIT) |
| 선임(td) | `unity-ankle` | [AnkleBreaker-Studio/unity-mcp-server](https://github.com/AnkleBreaker-Studio/unity-mcp-server) | Shader Graph·NavMesh·물리·빌드·프로파일링 (288도구) |
| QA(review·testeng) | `unity-hunt` | [HuntNight/unity-mcp-advanced](https://github.com/HuntNight/unity-mcp-advanced) | 스크린샷·플레이모드 제어·C# 실행·씬 분석 |

설치 절차(공통):
1. 각 저장소의 안내대로 **유니티 에디터 패키지**를 프로젝트에 설치하고 유니티를 켜 둡니다.
2. 그 저장소의 **MCP 서버 실행 명령**을 확인합니다 (uvx/node 등 저장소마다 다름).
3. `config/mcp.json`에서 해당 서버(`unity-coplay` 등)의 `command`/`args`를 그 명령으로 맞추고
   `"enabled": true` 로 바꿉니다. `{{UNITY}}`는 등록한 프로젝트 경로로 자동 치환됩니다.
4. 🔌 MCP 패널에서 **🔄 재연결** → 🟢 ready 확인.

이제 해당 팀 에이전트는 파일 읽기 대신 **에디터를 직접 조종**하며 리뷰·수정·검증합니다.

---

## 팁

- **먼저 파일 읽기 리뷰(1번)로 시작**하세요 — 설치가 거의 없고, 대부분의 코드/구조 보완점은 이걸로 나옵니다.
- 에디터 조종(2번)은 "실제로 돌려보고 고쳐야" 할 때만.
- 리뷰 품질은 모델 성능에 좌우됩니다. 개발 작업은 함수호출 지원 모델(현재 NVIDIA, 추후 로컬)이 필요합니다.
- 보안: `unity-project` 서버는 등록한 폴더 **밖**은 접근하지 못합니다(파일시스템 MCP의 allowed-dir 제약).
