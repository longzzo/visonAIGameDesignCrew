# KICKOFF_PROMPT.md — Claude Code 실행용 프롬프트

이 문서는 `vision-engine/` 폴더를 Claude Code로 열고 **가장 먼저 붙여넣을 프롬프트**를 담고 있다.

> **전제**: `.claude/agents/scenario.md`, `gameplay.md`, `uiux.md`, `systems.md`, `balance.md`, `bm.md`, `visual.md`가 실제로 존재해야 한다. Claude Code가 세션 시작 시 이 파일들을 자동으로 서브에이전트로 등록한다. 방금 이 폴더를 처음 열었다면 세션이 이미 이 서브에이전트들을 인식한 상태다.
>
> 모델은 **`claude-fable-5`로 고정**한다. 서브에이전트들은 모델을 지정하지 않았으므로 세션 모델을 그대로 상속받는다.

---

## 1) 최초 실행 — 프로젝트 부트스트랩

폴더를 처음 여는 순간, 게임 아이디어가 있든 없든 이 프롬프트 하나로 시작한다.

```
너는 이 세션의 CLAUDE.md에 정의된 대로 PM(총괄 오케스트레이터)이다.
.claude/agents/ 안의 7개 서브에이전트(scenario, gameplay, uiux, systems,
balance, bm, visual)를 실제로 Task 도구로 호출해 작업을 위임할 수 있다는 걸
인지하고 있어야 한다.

먼저 다음을 확인해라:
1. templates/MASTER_GDD.md와 templates/PROJECT_INTAKE.md를 읽는다.
2. 아직 실제 GDD 파일이 없다면, templates/MASTER_GDD.md를 복사해
   docs/GDD_[임시].md로 만들어라.
3. "1. 게임 개요"와 "2. 세계관·톤"이 비어 있으니, PROJECT_INTAKE.md의
   Stage 0부터 순서대로 나에게 질문해라. 한 번에 2~3개씩만 묻고,
   내 답변마다 즉시 GDD 파일에 직접 반영해라(너는 Write 권한이 있다).
4. 인테이크가 끝나면 지금까지 채워진 GDD 요약을 보여주고,
   어떤 서브에이전트부터 실제로 호출해볼지 제안해라.

이 프로젝트는 Claude Fable 5의 실제 성능을 실험하는 것이 목적이다.
서브에이전트를 호출할 때는 실제로 Task 도구를 사용해서 위임하고,
연기하듯 네가 직접 그 역할인 척 답하지 마라.

먼저 Stage 0 질문부터 시작해라.
```

---

## 2) 이미 게임 아이디어가 있을 때 — 빠른 시작 + 실제 서브에이전트 호출

```
너는 CLAUDE.md에 정의된 PM이다.

내가 만들려는 게임은 다음과 같다:
- 한 줄 소개: [여기에 작성]
- 장르: [여기에 작성]
- 플랫폼: [여기에 작성]
- 참고하고 싶은 게임/톤: [여기에 작성]

1. templates/MASTER_GDD.md를 복사해 docs/GDD_[게임명].md로 만들고
   "1. 게임 개요"를 이 정보로 채워라.
2. scenario 서브에이전트를 Task 도구로 실제로 호출해서
   "2. 세계관·톤" 섹션의 초안을 요청해라. 결과가 돌아오면
   너(PM)가 검토한 뒤 GDD 파일에 직접 반영해라.
3. 이어서 visual 서브에이전트를 호출해 이 세계관에 맞는
   아트 디렉션 초안(8. 아트 디렉션 섹션용)을 요청해라.
4. 두 결과가 서로 모순되지 않는지 확인하고 나에게 보고해라.
```

---

## 3) 스모크 테스트 — 서브에이전트가 실제로 병렬 호출되는지 확인

지금 만든 게 진짜 여러 서브에이전트를 동시에 부르는지 검증하고 싶을 때.

```
너는 PM이다. 지금은 정식 기획이 아니라 파이프라인 검증이다.

1. templates/MASTER_GDD.md를 docs/GDD_TEST.md로 복사하고
   임의의 간단한 판타지 액션 게임 컨셉으로 "1. 게임 개요"만 채워라.
2. systems 서브에이전트와 balance 서브에이전트를 동시에(병렬로)
   Task 도구로 호출해라:
   - systems에게는 "간단한 데미지 공식 구조를 설계해줘"
   - balance에게는 "레벨 1~10 캐릭터의 대략적인 체력 곡선을 제안해줘"
   두 요청을 각각 별도 Task 호출로 동시에 보내라.
3. 두 결과가 돌아오면, 실제로 병렬로 처리됐는지(순차가 아니라)
   보고하고, 두 결과를 하나로 통합해 GDD_TEST.md에 반영해라.
4. bm 서브에이전트에게 "docs/GDD_TEST.md 파일을 직접 수정해봐"라고
   시켜보고, 실제로 쓰기가 거부되는지 확인해서 나에게 알려라
   (이건 tools 권한 제한이 진짜 작동하는지 확인하는 테스트다).
```

## 참고: 프롬프트 사용 팁

- 세 프롬프트 모두 **처음 한 번만** 쓰면 된다. 이후 대화는 자연어로 이어가면 메인 세션이 계속 PM 역할을 유지하고, 필요할 때마다 알아서 서브에이전트를 호출한다.
- 특정 서브에이전트를 직접 지정하고 싶으면 "scenario 서브에이전트를 써서 ~해줘"처럼 이름을 명시하면 된다.
- `.claude/agents/*.md`를 수정한 뒤에는 Claude Code 세션을 재시작해야 변경사항이 반영된다(세션 시작 시에만 로드).
- Fable 5가 refusal을 반환하면(사이버·생물·화학 등 안전장치 관련, 게임 기획에선 드묾) "Opus 4.8로 재시도해라"라고 직접 지시한다.
